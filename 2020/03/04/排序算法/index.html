    写在前面

    查找和排序是算法的入门知识。一般在面试中最常考的是快速排序和归并排序，这两种排序的代码一定要很熟悉。还有插入排序、冒泡排序、堆排序、基数排序、桶排序等。面试官对于这些排序会要求比较各自的优劣、各种算法的思想及使用场景。还要分析时间和空间复杂度。通常查找和排序是面试的开始。

    一.冒泡排序

    冒泡排序是最简单的排序之一，如7，4，2，3，5，这五个数进行冒泡排序。首先从前向后冒泡，7和4比较，把7交换到后面，序列变成4，7，2，3，5。同理7和2交换，变成4，2，7，3，5。7和3交换，变成4，2，3，7，5。7和5交换，变成4，2，3，5，7。这样一次冒泡就完成了，把最大的数7排在最后。对剩下的序列依次冒泡就会得到一个有序序列。冒泡排序的复杂度为0（n^2）。

    class BubbleSort
{
    public:
    void swap(vector<int>& arr, int i, int j)
    {
        int tmp = arr[i];
        arr[i] = arr[j];
        arr[j] = tmp;
    }
    void bubbleSort(vector<int>& arr)
    {
        if(!arr.size()) return;
        for(int i = 0; i < arr.size() - 1; i++)
        {
            for(int j = 0; j < arr.size() - 1 -  i; j++)
            {
                if(array[j] > arr[j + 1])
                {
                    swap(arr, j+1, j);
                }
            }
        }
    }
}

    二.选择排序

    选择排序的思想和冒泡排序类似，都是在一次排序后把最小的元素放到最前面。但是过程不同。冒泡排序是通过响铃的比较和交换，而选择排序是通过对整体的选择。如7，4，2，3，5进行简单选择排序，首先选择7以外最小的数来和7交换，一次排序后就变成了2，4，7，3，5。对剩下的序列依次进行选择和交换，最终就会得到一个有序序列。选择排序可以看成冒泡排序的优化，因为它们的目的相同，只是选择在确定最小数的时候才交换，交换的次数大大减少。复杂度为O（n^2）

    class SlectSort
{
    public:
    void swap(vector<int>& arr, int i, int j)
    {
        int tmp = arr[i];
        arr[i] = arr[j];
        arr[j] = tmp;
    }
    void selectSort(vector<int>& arr)
    {
        if(!arr.size()) return;
        int minIndex = 0;
        for(int i = 0; i < arr.size() - 1; i++)
        {
            minIndex = i;
            for(int j = i + 1; j < arr.size() - 1 -  i; j++)
            {
                if(arr[j] < arr[minIndex])
                {
                    minIndex = j;
                }
            }
            if(i != minIndex)
            {
                swap(arr, i, minIndex);
            }
        }
    }
}

    三.插入排序

    插入排序不是通过交换位置而是通过比较找到合适的位置插入元素来达到排序的目的的。大家打扑克牌时，拿到一张牌怎么处理呢，当然是找到一个合适的位置插入啦，比如对7,4,2,3,5这个无序序列进行简单插入排序，首先假设第一个数的位置是正确的，就好比拿到第一张牌无需整理。然后第二张牌，4要插到7前面，把7后移一位，变成 4,7,2,3,5.然后7不动，2插到7前面，7后移一位；3插到4前面，5插到7前面，最后得到最终序列。插入排序的复杂度也为O(N^2).

    class InsertionSort
{
    public:
    void lnsertionSort(vector<int>& arr, int Left, int Right)           // left和right 分别表示要排序的左界和右界
    {
        if(Right - Left <=0 || Left <0 || Right > arr.size() - 1) return;
        for(int P = left + 1; P <= right; i++)
        {
            int tmp = arr[P]
            int j;
            for(j = P; j >0 && arr[j - 1] > tmp; j--)
            {
                arr[j] = arr[j - 1]
            }
            arr[j] = tmp;
        }
    }
}

    四.快速排序

    快速排序是实践中最快的已知排序算法。该算法之所以特别快，主要由于非常精炼且高度优化的内部循环。像归并排序一样，快速排序也是一种分治的递归算法。
    它由下面几步组成：

    1. 如果S中元素个数是0或1，则返回。（递归终止条件）
    2. 在集合S中选取枢纽元元素v、
    3. 将S-{V}（S中其余的元素）分成两个不想交的集合：S1：小于等于v的部分和S2：大于等于v的部分
    4. 返回{quicksort（S1），v， quicksort（S2）}。（递归思想）

    举个例子：对7,4,2,3,5这个无序序列进行快速排序，思路是右指针找比枢纽元小的，左指针找比基准数大的，找到后交换左右指针的数。
    这里用最右边的5作为枢纽元，最终会把5小的移动到5的左边，比5大的移动到5的右边
    7，4，2，3，5 首先设置i，j两个指针分别指向两端，i指针先扫描（思考一下为什么？）7比5大停止，然后扫描j，3比5小停止。交换i，j所指向数据的位置。
    3，4，2，7，5 然后指针再扫描，这时i扫描到7停止，j扫描到2停止，此时i，j交错，故不再交换，分割的最后一步是将枢纽元与i所指向的元素交换
    3，4，2，5，7 一次划分后达到了左边比5小，右边比5大的目的。之后对左右子序列递归排序，最终得到有序序列。
    上面留下来一个问题，为什么要i指针先扫描，首先，这也不是绝对的，这取决于枢纽元的位置，因为在指针相遇时，要交换枢纽元和源于位置的元素。这里选取右边的数作为枢纽元，那么相遇的数要比枢纽元大，所以i指针险以动才能确保可以找到比枢纽元大的数。
    快速排序是不稳定的，它的平均时间复杂度是O（N logN），最坏性能是O（N^2）,但是稍加努力可以避免这种最坏情形。

    对于总是取最右边元素作为枢纽元的办法，可能会产生O（N^2）的最坏性能, 这在数据量比较大的情况下是不愿意看到的，因此，有一种办法是将arr[Left]、arr[Right]、arr[Center]适当排序，即该三元素中的最小者被分在arr[Left], 而这正是分割阶段应该将它放到的位置。三元素中的最大者被分在arr[Right], 这也是正确的位置，因为它大于枢纽元。因此，我们可以把枢纽元放到arr[Right - 1]并在分割阶段将i 和 j 初始化到left + 1 和 Right - 2. 因为arr[left]比枢纽元小，所以将它用作j 的警戒标记（不用担心j越界），这是另一个好处。因为i会停在那些等于枢纽元的位置，所以将枢纽元存储在arr[Right - 1]可以提供一个境界标记。

    class QuickSort 
{
    public:
    #define Cutoff (3)

    int Median(vector<int> &arr, int left, int right)
    {
        int Center = (Left + Right) / 2;

        if(arr[Left] > arr[Center]) )
            swap(arr[left], arr[Center]);
        if(arr[Left] > arr[Right])
            swap(arr[left], arr[Right]);
        if(arr[Center] > arr[Right])
            swap(arr[Center], arr[Right]);

        swap(arr[Center], arr[Right - 1]);
        return arr[Right - 1];
    }
    void Qsort(vector<int> & arr, int left, int right)
    {
        int i, j;
        int Pivot;
        if(left + cutoff <= Right)
        {
            Pivot = Median(arr, Left, Right);
            i = Left;
            j = Right - 1;
            for( ; ; )
            {
                while(arr[++i] < pivot){}
                while(arr[--j]) < pivot{}
                if(i < j)
                    swap(arr[i], arr[j]);
                else break;
            }
            swap(arr[i], arr[Right - 1]);

            QSort(arr, left, i - 1);
            QSort(arr, i + 1, right);

        else

    五.归并排序

    归并排序以O（NlogN）最坏情形运行时间运行，而所使用的比较次数几乎是最优的。它是递归算法一个很好的实例。
    这个算法中基本的操作是合并两个已排序的表。因为这两个表是已排序的，所以若将输出放到第三个表中，则该算法可以通过对输入数据一趟排序来完成。基本的合并算法是去两个输入数组A和B，一个输出数组C，以及三个计数器Aptr、Bptr、Cptr，他们初始置于对应数组的开始端。A[Aptr]和B[Bptr]中的较小者被拷贝到C中的下一个位置，相关的计数器向前推进一步。当两个输入表有一个用完的时候，则将另一个表中的剩余部分拷贝到C中。

    Merge例程是精妙的。如果对Merge的每个递归调用均局部声明一个临时数组，那么在任艺术课就可能有logN个临时数组处于活动期，这对于小内存机器是致命的。
    严密测试之处，由于Merge位于Msort的最后一行，因此在任意时刻只需要一个临时数组活动，而且可以使用该临时数组的任一部分。

    void MSort(int A[], int TmpArray[], int left, int right)
{
    int Center;

    if(left < right)
    {
        Center = (left + right) / 2;
        MSort(A, TmpArray, left, Center);
        MSort(A, TmpArray, Center + 1; right);
        Merge(A, TmpArray, left, Center + 1, right);
    }
}

void Mergesort(int A[], int N)
{
    int * TmpArray = new int[N];

    if(TmpArray != NULL)
    {
        MSort(A, TmpArray, 0, N - 1;)
        delete TmpArray;
    }
    else 
        FataError(" No space for tmp array!!!");

void Merge(int A[], int TmpArray[], int Lpos, int Rpos, int RightEnd)
{
    int i, LeftEnd, NumElements, TmpPos;

    LeftEnd = Rpos - 1;
    TmpPos = Lpos;
    NumElements = RightEnd - Lpos + 1;

    while(Lpos <= LeftEnd && Rpos <= RightEnd)
        if(A[Lpos] <= A[Rpos])
            TmpArray[TmpPos++] = A[Lpos++];
        else
            TmpArray[TmpPos++] = A[Rpos++];

    while(Lpos <= LeftEnd)
        TmpArray[TmpPos++] = A[Lpos++];
    while(Rpos <= RightEnd)
        TmpArray[TmpPos++] = A[Rpos++];

    for(i = -; i < NumElements; i++, RightEnd--)
        A[RightEnd] = TmpArray[RightEnd];
}

    六.堆排序

    简单总结下堆排序的基本思路：
    a. 将无序序列构建成一个堆，根据升序降序需求选择大顶堆或小顶堆
    b. 将堆顶元素与末尾元素交换，将最大元素沉到数组末端
    c.重新调整结构，使其满足堆定义，然后继续交换堆顶元素与当前末尾元素，反复执行调整+交换步骤，知道整个序列有序

    代码：

    class HeapSort{
public:
    void hSort(vector<int>& nums)
    {

        //1.构建大顶堆
        for(int i = nums.size() / 2 - 1; i >= 0; i--)

        {

 //2.调整堆结构+交换堆顶元素与末尾元素
